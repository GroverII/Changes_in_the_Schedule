Серверная часть.
Для запуска зайти в кансоль и прописать 
npm start
или запустить 
start.bat

server.js - весь код сервера. Чтобы изменить порт нужно в строке 
const port = *порт*;
заменить *порт* на число. К примеру так -
const port = 3001;
Для того, чтобы изменения вступили в силу нужно перезапустить сервер. Находясь в консоли нажимаем ctrl+c и пишем 'y' или 'n'. Нажимаем энтер. Прописываем снова npm start. 
Можно просто закрыть кансоль и запустить. Но так делать не рекомендую.

updates.json - файл для проверки обновлений настроек скрипта(связь с эксель таблицей). Меняется автоматически при изменении настроек сервера.

ignoreIt.png - коды для использования reCAPTCHA. Пока не требовались нигде.

conf.txt - конфигурации сервера:
   csharpScriptPath - путь до скрипта. На данный момент изменить нельзя через админменю.
   credentialsPath - путь до секретного ключа для использования spreadsheets. Частично можно изменить в настройках скрипта. Пока невозможно изменить полностью.
   linksFilePath - путь до папки, где будут все папки с файлами на вывод клиентской стороны. На данный момент изменить нельзя через админменю.
   links - выбранная папка на данный момент. На данный момент изменить нельзя через админменю.
   TimeZone - зона времени для стороны клиента.  На данный момент изменить нельзя через админменю.

8c6976e5b5410415bde908....txt - файл может иметь другое название, но подобную рандомность. Представляет собой зашифрованные логин и пароль. Логин зашифрован в названии, пароль внутри.Можно сделать копию чтобы заменить в случае потери логина или пароля. При необходимости можно добавить сразу несколько таких файлов. Будут несколько пользователей. Админ может изменить через админменю.

uploads - папка, где хроняться временно последние добавленные файлы. Файлы внутри можно удалять, но они должны подчищаться автоматически со временем. Изменяется, когда на сервер закидывают файлы(видео, фото, ...).

SheetsInfo - папка со скриптом. Возвращает лист данных из spreadsheets. Не меняется(за исключением конфигураций).
Больше информации в самой папке.

dir's - папка, где храняться папки с файлами. В папках всегда должен быть файл order.json(создаётся аввтоматически), по которому и выбирается порядок появления файлов на клиенте. Меняется при добавлении новых файлов в эту папку или при изменении порядка(только файл order.json).
